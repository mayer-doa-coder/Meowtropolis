# Baseline Screenshots (Phase 0)

## Purpose
Capture the current UI baseline before any redesign or refactor.

## Capture Rules
- Use one consistent simulator device for all screenshots.
- Use one consistent appearance mode for all screenshots.
- Do not enable debug overlays.
- Capture from stable app states (no transient alerts unless documenting an issue).

## Required Screens
1. Launch screen
2. Login
3. Signup
4. Dashboard (main tab)
5. Pet Profile (list)
6. Pet Profile (add or edit)
7. Grooming flow
8. Vet consultation flow
9. Marketplace list
10. Marketplace cart (if present in current flow)

## Naming Convention
Use *_v1.png naming and store all images in this folder.

- launch_screen_v1.png
