# Phase 0 Guardrails (Blocking)

## Non-Negotiable Constraints

Do NOT change in Phase 0:
- Font families and typography family usage
- Color theme values defined by AppDesign
- Firebase/backend connections and service contracts
- Navigation structure (TabView, NavigationStack)

If any requested change risks these areas:
- Stop implementation
- Log the request and risk in docs/issue_inventory.md
- Defer decision to team review

## Allowed Work in Phase 0
- Add guardrail comments
- Capture baseline screenshots
- Document and categorize issues
- Prepare checklists and team alignment artifacts

## Not Allowed in Phase 0
- Bug fixes
- Refactors
- New features
- UI redesign implementation

## Team Alignment Checklist
- [ ] All members reviewed guardrails
- [ ] All members agree Phase 0 is documentation/observation only
- [ ] Screenshot capture owner assigned
- [ ] Issue triage owner assigned
- [ ] Phase 0 exit criteria reviewed
